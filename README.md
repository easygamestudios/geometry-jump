# Geometry Jump

Приложение для macOS: игра + редактор уровней. Внутри — та же игра на HTML5/Canvas,
что и раньше, но завёрнутая в Electron, поэтому запускается как обычная программа,
а не как сайт в браузере.

## Запустить

```
npm start
```

## Собрать приложение

```
npm run build
```

Готовое лежит в `dist/`:

- `Geometry Jump-1.2.0-arm64.dmg` — образ для установки
- `mac-arm64/Geometry Jump.app` — само приложение, можно сразу перетащить в «Программы»

## Проверить, что всё живо

```
npx electron electron/verify.js          # лобби
npx electron electron/verify.js editor   # редактор
npx electron electron/verify.js export   # редактор + экспорт файла уровня
npx electron electron/verify.js play     # запуск уровня, проверка что игрок едет
```

Скрипт грузит игру, собирает ошибки консоли, снимает кадр в `dist-verify/`
и возвращает ненулевой код, если что-то сломалось.

## Что где

```
app/            — сама игра (перенесена из Geometry web без изменений логики)
  dev.html      — то, что открывает приложение: игра + редактор
  index.html    — вариант без редактора, если захочешь выложить в веб
  js/engine.js  — движок: физика, коллизии, отрисовка, звук
  js/main.js    — лобби, гараж, выбор уровня, карта, HUD
  js/editor.js  — редактор уровней
  js/levels.js  — встроенные уровни
  music/        — музыка уровней
  textures/     — кубики
electron/main.js    — оболочка: окно, меню, протокол gjump://, диалог сохранения
electron/verify.js  — проверочный прогон
build/icon.icns     — иконка приложения
```

## Как это устроено

Игра грузится не через `file://`, а через собственный протокол `gjump://game/`.
Так у страницы нормальное происхождение, и `localStorage` — прогресс уровней,
настройки, автосохранение редактора — переживает перезапуск приложения.

Кнопка **⤓ Экспорт** в редакторе открывает системный диалог сохранения
(`electron/main.js`, обработчик `will-download`), файл `.gwlevel.json`
кладётся куда скажешь. Импорт и загрузка музыки работают через обычный
системный выбор файла.

## Если передаёшь приложение другому человеку

Приложение подписано ad-hoc — то есть без сертификата Apple. На твоём компьютере
оно запускается без вопросов. У того, кто скачает `.dmg`, macOS при первом
запуске скажет:

> Не удалось проверить, что программа «Geometry Jump» не содержит вредоносного ПО.

Как открыть (macOS 15 и новее):

1. Перетащить приложение в «Программы» и запустить — появится предупреждение, нажать «Готово»
2. Системные настройки → «Конфиденциальность и безопасность» → пролистать вниз
3. Там будет строка про заблокированное приложение → «Открыть всё равно» → пароль или Touch ID

Способ с правой кнопкой → «Открыть» больше не работает: Apple убрала его начиная
с macOS 15 Sequoia.

Одной командой в Терминале — то же самое, но без предупреждений вообще:

```
xattr -dr com.apple.quarantine "/Applications/Geometry Jump.app"
```

Чтобы приложение открывалось у всех сразу и без плясок, нужен платный
Apple Developer ID и нотаризация.

### Почему подпись вообще делается

`electron-builder` не подписывает бандл, если нет сертификата, и оставляет
linker-подпись без ресурсной печати. С такой подписью macOS говорит не
«не удалось проверить разработчика», а **«приложение повреждено»** — а из
«повреждено» кнопки «Открыть всё равно» уже нет, и обойти это обычным путём
нельзя. Поэтому `electron/afterPack.js` подписывает приложение ad-hoc
до сборки `.dmg`.

Исходная веб-версия осталась нетронутой в `~/Desktop/Geometry web`.
